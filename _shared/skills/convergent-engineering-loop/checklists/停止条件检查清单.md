# 停止条件检查清单

## 成功停止

满足任一条件即可停止：

- 误差降为零；
- 验收标准满足；
- 目标测试通过；
- 相关检查通过；
- 文档与实现一致；
- Review 阻塞评论已解决；
- Harness 可稳定运行。

## 安全停止

出现以下情况应停止并请求用户确认：

- 缺少关键信息；
- 需求存在歧义；
- 验证环境不可用；
- 涉及高风险修改；
- 存在多个合理方案需要取舍；
- 需要凭证、权限或外部服务。

## 失败或震荡停止

出现以下情况应停止并总结：

- 多轮没有改进；
- 修复 A 破坏 B，修复 B 又破坏 A；
- 状态在两个方案之间循环；
- 每次修改都让误差上升；
- Agent 无法找到更小的安全动作。
