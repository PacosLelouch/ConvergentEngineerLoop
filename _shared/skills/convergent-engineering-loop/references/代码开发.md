# 代码开发收敛参考

当任务涉及功能实现、Bug 修复、构建错误、运行错误、类型错误、Lint 错误、代码重构时，使用本参考。

## 误差度量

可使用：

$$
E_{\text{dev}}=
w_1F_{\text{test}}
+w_2F_{\text{build}}
+w_3F_{\text{type}}
+w_4F_{\text{lint}}
+w_5F_{\text{runtime}}
$$

其中：

- $F_{\text{test}}$：失败测试数量；
- $F_{\text{build}}$：构建失败数量；
- $F_{\text{type}}$：类型错误数量；
- $F_{\text{lint}}$：Lint 错误数量；
- $F_{\text{runtime}}$：运行时错误数量。

## 迭代规则

每轮只做一个最小代码修改。

| 收敛目标 | 每轮动作 |
|---------|---------|
| 目标功能可用 | 修复一个失败簇 |
| 构建通过 | 修改最小相关代码路径 |
| 相关测试通过 | 为新行为添加聚焦测试 |
| 没有新增关键错误 | 先修根因，再修表象 |
| Diff 范围有限且可解释 | 保持公共接口稳定 |

避免：

- 重写无关文件；
- 全仓库格式化；
- 删除测试以通过测试；
- 削弱断言；
- 无理由引入新依赖；
- 把重构和功能改动混在一起。

## 验证方式

推荐验证顺序：

```
目标测试 → 相关测试文件 → 类型检查 → Lint → 完整测试套件 → 构建
```

## 回滚条件

应回滚或重做的条件：

- 失败数量增加；
- 修改了无关行为；
- 只是隐藏错误而非修复错误；
- 无法解释为什么该修改有效。
