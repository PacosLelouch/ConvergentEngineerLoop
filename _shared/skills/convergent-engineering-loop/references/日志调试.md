# 日志调试收敛参考

当任务涉及日志分析、堆栈追踪、线上问题、复现步骤、根因分析、运行时异常时，使用本参考。

## 误差度量

可使用：

$$
E_{\text{debug}}=
w_1H+w_2U+w_3R+w_4N
$$

其中：

- $H$：活跃假设数量；
- $U$：未知关键信息数量；
- $R$：复现不确定性；
- $N$：噪声或无关线索数量。

## 迭代规则

每轮应缩小假设空间。

| 收敛目标 | 每轮动作 |
|---------|---------|
| 问题可复现或边界清晰 | 缩小复现条件 |
| 根因明确 | 提取第一个有意义错误 |
| 已排除假设有记录 | 排除一个原因 |
| 修复方案与根因对应 | 补充一个关键事实 |
| 验证结果确认问题被解决 | 验证一个假设 |

避免：

- 无证据猜测；
- 重复尝试已失败方案；
- 一次改变多个变量；
- 把症状当根因；
- 忽略环境差异。

## 必用格式

```
已知事实：
未知信息：
活跃假设：
已排除假设：
下一步实验：
预期观察：
实际结果：
更新后的结论：
```

## 验证方式

检查：

- 活跃假设是否减少？
- 是否获得新事实？
- 问题是否更容易复现？
- 修复是否直接对应根因？
