# 架构设计收敛参考

当任务涉及系统架构、模块拆分、接口定义、数据流、依赖关系、技术选型、ADR 时，使用本参考。

## 误差度量

可使用：

$$
E_{\text{arch}}=
w_1U+w_2I+w_3D+w_4K+w_5P
$$

其中：

- $U$：未定义模块数量；
- $I$：接口不清晰项数量；
- $D$：不合理依赖或循环依赖数量；
- $K$：关键决策缺失数量；
- $P$：未解决的性能、安全、扩展性风险数量。

## 迭代规则

每轮应让架构更明确、更稳定。

| 收敛目标 | 每轮动作 |
|---------|---------|
| 模块职责清晰 | 明确一个模块边界 |
| 接口边界明确 | 定义一个接口 |
| 关键决策已记录 | 记录一个 ADR |
| 依赖关系可解释 | 消除一个不合理依赖 |
| 已知风险有处理策略 | 处理一个架构风险 |

避免：

- 每轮更换技术栈；
- 每轮重画整个架构；
- 无理由增加抽象层；
- 未做影响分析就修改已冻结接口；
- 同时改动多个核心架构决策。

## 验证方式

检查：

- 实现是否因此更容易开始？
- 模块之间职责是否更清晰？
- 决策是否可追溯？
- 是否引入新的耦合或隐藏复杂度？
