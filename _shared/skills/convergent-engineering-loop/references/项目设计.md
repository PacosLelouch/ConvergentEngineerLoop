# 项目设计收敛参考

当任务涉及需求澄清、项目范围、目标定义、约束条件、风险分析、验收标准时，使用本参考。

## 误差度量

可使用：

$$
E_{\text{design}}=
w_1Q+w_2A+w_3C+w_4S+w_5R
$$

其中：

- $Q$：未解决问题数量；
- $A$：未验证假设数量；
- $C$：冲突需求数量；
- $S$：范围不清晰事项数量；
- $R$：未缓解风险数量。

## 迭代规则

每轮应减少需求不确定性。

| 收敛目标 | 每轮动作 |
|---------|---------|
| 关键问题已回答或明确延后 | 关闭一组相关未决问题 |
| 冲突需求已解决 | 解决一个需求冲突 |
| 验收标准可测试 | 将一个模糊需求转成可测试验收标准 |
| 关键假设已记录 | 标记一个假设为已确认、已否定或待验证 |
| 非目标明确 | 明确一个非目标 |

避免：

- 无理由新增范围；
- 每轮重写整个项目愿景；
- 尚未澄清需求就进入实现细节；
- 把假设写成事实。

## 验证方式

检查：

- 开发者是否可以根据该设计开始实现？
- 测试人员是否可以根据验收标准验证？
- 用户是否能判断做完与否？
- 未决问题是否比上一轮减少？

## 交叉验证

当计划文档包含实现状态标记（done/undone/partial）时，标记准确度的误差维度（$M_{\text{over}}$、$M_{\text{under}}$、$M_{\text{quality}}$）定义见 `references/交叉验证.md`。
