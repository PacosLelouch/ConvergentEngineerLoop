# 文档编写收敛参考

当任务涉及 README、API 文档、配置文档、示例代码、变更日志、迁移指南、架构说明时，使用本参考。

## 误差度量

可使用：

$$
E_{\text{doc}}=
w_1O+w_2M+w_3B+w_4G+w_5S
$$

其中：

- $O$：过期内容数量；
- $M$：缺失说明数量；
- $B$：不可运行示例数量；
- $G$：术语不一致数量；
- $S$：结构或可读性问题数量。

## 迭代规则

每轮只改进文档中的一个明确部分。

| 收敛目标 | 每轮动作 |
|---------|---------|
| 命令准确 | 更新一个过期段落 |
| 示例可运行或明确说明为伪代码 | 修复一个不可运行示例 |
| API 与当前代码一致 | 对齐一个 API 文档与代码签名 |
| 配置项说明完整 | 增加一个缺失说明 |
| 术语一致 | 保持原有文档结构 |

避免：

- 无必要重写整篇文档；
- 编造代码不支持的行为；
- 加入未验证命令；
- 无理由更换术语；
- 把不稳定实现细节写成公开 API。

## 验证方式

检查：

- 文档中的命令是否存在？
- 示例是否匹配当前接口？
- 新用户能否按步骤完成操作？
- 修改部分是否与上下文一致？
- 文档是否减少了不确定性？
