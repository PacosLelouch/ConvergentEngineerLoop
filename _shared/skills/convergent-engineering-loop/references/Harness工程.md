# Harness 工程收敛参考

当任务涉及 CI、评测 Harness、Benchmark 脚本、可复现环境、自动化流程、测试运行器、Agent 执行 Harness 时，使用本参考。

## 误差度量

可使用：

$$
E_{\text{harness}}=
w_1F+w_2V+w_3R+w_4N+w_5C
$$

其中：

- $F$：Harness 失败数量；
- $V$：版本漂移数量；
- $R$：不可复现结果数量；
- $N$：非确定性因素数量；
- $C$：配置不一致数量。

## 迭代规则

每轮应让 Harness 更确定、更可观测或更可复现。

| 收敛目标 | 每轮动作 |
|---------|---------|
| 可以一键运行 | 修复一个失败 Job |
| 依赖版本固定 | 固定一个不稳定依赖 |
| 失败原因可诊断 | 区分 Harness 失败和产品失败 |
| 结果可复现 | 增加一个环境检查 |
| 输出格式稳定 | 改进报告可读性 |

避免：

- 在 Harness 中隐藏真实失败；
- 为了让 CI 通过而跳过重要测试；
- 依赖未固定的外部状态；
- 混合评测逻辑和业务逻辑；
- 一次修改多个基础设施层。

## 验证方式

检查：

- 是否能在干净环境运行？
- 依赖是否固定？
- 随机种子是否受控？
- 失败输出是否可行动？
- CI 与本地是否一致？

## 接受条件

接受修改的条件：

- 目标失败减少或消失；
- 没有引入更高优先级回归；
- 修改范围可解释；
- 验证结果支持修改。
