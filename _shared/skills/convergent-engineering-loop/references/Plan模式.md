# Plan 模式收敛参考

当 CEL 技能在 CodeBuddy Plan 模式下运行时，使用本参考。

## Plan 模式与 CEL 的映射

| Plan 阶段 | 状态 | 对应 CEL 轮次 | 说明 |
|-----------|------|-------------|------|
| 需求澄清 | prepare | 评估轮次 | 识别当前状态、定义目标、计算初始误差 |
| 方案制定 | prepare | 评估轮次 | 细化误差度量、确定参考文件 |
| 方案确认 | ready | 评估轮次（最终） | 输出评估报告供用户审阅 |
| 方案实施 | building | 修改轮次 | 按 todo 逐项执行最小更新 |
| 方案完成 | finished | — | 停止条件满足 |

## todo→迭代映射

- 每个 todo 项可包含**多轮** CEL 迭代，直到该 todo 对应的误差降到零或可接受阈值
- todo 的完成标准 = CEL 的停止条件（误差为零、验收标准满足等）
- 当一个 todo 完成后，CEL 持续迭代规则要求自动进入下一个 todo（而非停止整个任务）

## 依赖感知

Plan 模式的 todo 通常有依赖顺序。CEL 步骤5"选择最小更新"时：

- 不选择前置 todo 未完成的项
- 如果当前 todo 的前置 todo 存在非零误差，应先处理前置 todo
- 如果当前 todo 无法推进（前置依赖阻塞），属于安全停止，应暂停并报告

## 评估轮次输出→Plan 准备内容

评估轮次的输出可直接作为 Plan 模式准备阶段的内容：

| CEL 评估输出 | Plan 对应内容 |
|-------------|-------------|
| 当前状态 | 需求分析中的现状描述 |
| 目标状态 | 技术方案中的完成条件 |
| 误差度量 | 任务列表中各 todo 的验收标准 |
| 主要误差来源 | 任务列表中优先级排序的依据 |
| 差异清单 | 方案中的待解决问题 |

## 停止条件统一

| CEL 停止条件 | Plan 模式行为 |
|-------------|-------------|
| 成功停止 | 标记 todo 为 completed，自动进入下一个 |
| 安全停止 | 暂停 Plan（状态不变），请求用户确认 |
| 失败或震荡停止 | 暂停 Plan，报告冲突，请求用户决策 |

## 误差度量

可使用：

$$
E_{\text{plan}}=
w_1U+w_2D+w_3B
$$

其中：

- $U$：未完成 todo 数量；
- $D$：依赖阻塞的 todo 数量；
- $B$：build/验证失败的 todo 数量。

## 迭代规则

每轮应推进一个 todo 向完成靠近。

| 收敛目标 | 每轮动作 |
|---------|---------|
| 所有 todo 完成 | 选择一个可执行的 todo，执行其对应的最小更新 |
| 依赖无阻塞 | 完成前置 todo，解除后续阻塞 |
| 验证全部通过 | 为当前 todo 运行验证，确认误差下降 |

避免：

- 跳过前置依赖直接执行后续 todo；
- 一个 todo 未完成就切换到另一个无关 todo；
- 在 building 状态下执行评估操作（应先完成当前修改轮次）。

## 验证方式

检查：

- 当前 todo 的验收标准是否满足？
- 是否引入了新问题导致其他 todo 误差上升？
- 前置依赖是否仍然满足？
- Plan 整体进度是否在推进？

## 分批执行

当 Plan 的 todo 数量较多或单项复杂度较高时，应分批执行。详见 `references/分批迭代.md`。

Plan 模式分批要点：
- 按依赖顺序分批，每批 3-5 个 todo
- 依赖链上的 todo 应在同一批次
- 可独立执行的 todo 可灵活分配到不同批次

### Codex Plan→目标模式

当在 Codex 环境下使用 Plan 模式完成规划后，可将 Plan 的输出转化为目标模式提示词，交由 Codex 目标模式自主执行。详见"目标模式提示词生成"章节。

## 目标模式提示词生成

当在 Codex Plan 模式下完成规划后，可按以下规则生成目标模式提示词，实现从规划到自主执行的桥接。

### 生成时机

- Plan 进入 ready 状态（方案确认）后
- 用户确认方案并要求开始执行时

### 生成规则

1. 从 Plan 的评估轮次输出中提取：目标状态、误差度量、参考文件
2. 从 todo 列表中提取：任务列表、依赖顺序、验收标准
3. 从 batch 划分中提取：批次计划（如已分批）
4. 从编排 Agent 传递的上下文中提取：约束条件（如有）
5. 按 `templates/目标模式提示词.md` 模板格式输出

### 注意事项

- 提示词应包含完整的目标描述，不依赖外部上下文
- 验收标准必须明确、可验证
- 依赖顺序必须完整列出
- 安全约束必须显式标注
