# 测试工程收敛参考

当任务涉及失败测试、测试生成、覆盖率提升、Flaky 测试、回归测试、测试策略时，使用本参考。

## 误差度量

可使用：

$$
E_{\text{test}}=
w_1F+w_2G+w_3K+w_4D+w_5R
$$

其中：

- $F$：失败测试数量；
- $G$：覆盖率缺口；
- $K$：Flaky 测试数量；
- $D$：重复或低价值测试数量；
- $R$：回归风险数量。

## 迭代规则

每轮只改进一个测试目标。

| 收敛目标 | 每轮动作 |
|---------|---------|
| 目标测试通过 | 修复一个失败测试簇 |
| 关键行为有覆盖 | 覆盖一个未覆盖分支 |
| 已修复 Bug 有回归测试 | 添加一个聚焦回归测试 |
| Flaky 测试被隔离或修复 | 消除一个不稳定因素 |
| 测试确定、可复现、有意义 | 简化一个脆弱测试 |

避免：

- 削弱断言；
- 删除有效失败测试；
- 添加目的不明的大而全测试；
- 依赖时间、网络或不稳定外部服务；
- 不看语义变化就更新 snapshot。

## 验证方式

推荐运行：

```
目标测试 → 受影响测试文件 → 受影响模块覆盖率 → 必要时运行完整测试套件
```

## 接受条件

接受修改的条件：

- 测试正确性、稳定性或覆盖率提升；
- 没有隐藏真实失败；
- 新增测试能说明具体行为；
- 验证结果可复现。
